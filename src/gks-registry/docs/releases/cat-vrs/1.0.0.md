# cat-vrs 1.0.0

> Released: 2025-06-11

## Release Notes

# Cat-VRS 1.0.0 Official Release

We are excited to announce the first official release of Cat-VRS, providing a robust foundation for categorical variation representation within the GA4GH GKS ecosystem.

## Major Features

- **Support for Categorical Variation**: Comprehensive modeling of categorical variation, including support for hyperintensional set objects and new requirements for `CategoricalVariation` naming.
- **Expanded and Improved Documentation**: Full decision record, product roadmap, and extensive examples have been integrated into the documentation, including specification examples for complex variants like BRAF V600 and TP-53 CNLoss.
- **Example Expansion**: Significantly expanded set of examples to aid implementers, including canonical alleles, categorical CNVs, protein sequence consequences, and described variants.
- **Improved Specification and Comments**: Updated comments and annotations throughout the code and specification to clarify intent and enhance readability.
- **Infrastructure and Quality**: Updates to support VRS 2.0, improved internal and external hyperlinking, and enhancements to citation formats and design documentation.

## Get Started

For comprehensive documentation, examples, and the product roadmap, please visit our [ReadTheDocs](https://cat-vrs.ga4gh.org/) site.

The protein sequence consequence, like a CanonicalAllele, is defined by an Allele that is representative of a collection of congruent Protein Alleles that share the same altered codon(s). |
