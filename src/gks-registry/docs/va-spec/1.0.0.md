## Release Notes

# VA-Spec 1.0.0 Official Release

The GA4GH Variant Annotation Specification (VA-SPEC) 1.0.0 is now released! This milestone establishes a robust standard for representing variant annotation data in genomic research and clinical applications.

## Major Changes Over Ballot Cycle

- **Finalized Variant Annotation Model**: Comprehensive schema for representing variant annotation, including support for study results, source datasets, and profile metadata.
- **Documentation and ReadMe Updates**: Documentation has been thoroughly updated for v1, including class/attribute descriptions and improved onboarding information.
- **Schema and Metadata Enhancements**:
  - Type of the `pmid` field changed from integer to string for broader compatibility.
  - All `profile-source.yaml` files merged into a single unified source.
  - Refined metadata for improved clarity and consistency.
- **Release Preparation and Housekeeping**:
  - Merged all ballot updates for the 2025-03 cycle.
  - Streamlined documentation build process and fixed ReadTheDocs branch handling.

## Getting Started

For comprehensive documentation and schema details, please visit the [ReadTheDocs](https://va-spec.ga4gh.org/) site.

| [ConditionSet](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/ConditionSet) | trial use | A set of conditions (diseases, phenotypes, traits). A set of two or more conditions that co-occur in the same patient/subject, or are manifest individually in a different subset of participants in a research study. |
| [Contribution](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/Contribution) | trial use | An action taken by an agent in contributing to the creation, modification, assessment, or deprecation of a particular entity (e.g. a Statement, EvidenceLine, DataSet, Publication, etc.) |
| [DataSet](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/DataSet) | trial use | A collection of related data items or records that are organized together in a common format or structure, to enable their computational manipulation as a unit. |
| [Document](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/Document) | trial use | A collection of information, usually in a text-based or graphic human-readable form, intended to be read and understood together as a whole. |
| [EvidenceLine](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/EvidenceLine) | trial use | An independent, evidence-based argument that may support or refute the validity of a specific Proposition. The strength and direction of this argument is based on an interpretation of one or more pieces of information as evidence for or against the target Proposition. |

| [Method](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/Method) | trial use | A set of instructions that specify how to achieve some objective. |
| [Statement](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/Statement) | trial use | A claim of purported truth as made by a particular agent, on a particular occasion. Statements may be used to put forth a possible fact (i.e. a 'Proposition') as true or false, or to provide a more nuanced assessment of the level of confidence or evidence supporting a particular Proposition. |
| [StudyGroup](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/StudyGroup) | trial use | A collection of individuals or specimens from the same taxonomic class, selected for analysis in a scientific study based on their exhibiting one or more common characteristics  (e.g. species, race, age, gender, disease state, income). May be referred to as a 'cohort' or 'population' in specific research settings. |
| [StudyResult](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/StudyResult) | trial use | A collection of data items from a single study that pertain to a particular subject or experimental unit in the study, along with optional provenance information describing how these data items were generated. |
| [SubjectVariantProposition](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/SubjectVariantProposition) | trial use | A Proposition that has a variant as the subject. |
| [Therapeutic](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/Therapeutic) | trial use | An individual therapy (drug, procedure, behavioral intervention, etc.), or group of therapies (TherapyGroup). |
| [TherapyGroup](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/TherapyGroup) | trial use | A group of two or more therapies that are applied in combination to a single patient/subject, or applied individually to a different subset of participants in a research study. |
| [TumorVariantFrequencyStudyResult](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/TumorVariantFrequencyStudyResult) | draft | A Study Result that reports measures related to the frequency of a variant in a particular tumor type. |
| [VariantDiagnosticProposition](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/VariantDiagnosticProposition) | trial use | A Proposition about whether a variant is associated with a disease (a diagnostic inclusion criterion), or absence of a disease (diagnostic exclusion criterion). |
| [VariantDiagnosticStudyStatement](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/aac-2017/json/VariantDiagnosticStudyStatement) | draft | A statement reporting a conclusion from a single study about whether a variant is associated with a disease (a diagnostic inclusion criterion), or absence of a disease (diagnostic exclusion criterion) - based on interpretation of the study's results. |
| [VariantOncogenicityEvidenceLine](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/ccv-2022/json/VariantOncogenicityEvidenceLine) | draft | An Evidence Line that describes how evidence for a variant was interpreted to determine if a specific CCV 2022 criterion code is met, and the strength that evidence this provides for or against the variant's oncogenicity. In the CCV Framework, evidence is assessed by determining if a specific criterion (e.g. 'OM2') with a default strength (e.g. 'moderate') is 'met' or 'not met', and in some cases adjusting the default strength based on the quality and abundance of evidence. |
| [VariantOncogenicityProposition](https://github.com/ga4gh/va-spec/blob/1.0.0/schema/va-spec/base/json/VariantOncogenicityProposition) | trial use | A proposition describing the role of a variant in causing a tumor type. |
